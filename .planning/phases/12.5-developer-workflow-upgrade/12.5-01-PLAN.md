# Plan 12.5-01: Warp Terminal Setup

## Objective

Install Warp Terminal on Windows, configure shell integration, and create Samsara-specific launch configurations for efficient multi-pane development.

## Tasks

### 1. Install Warp Terminal

- Install via `winget install Warp.Warp`
- Launch and complete account setup / authentication
- Verify PowerShell and Git Bash shell support

### 2. Configure Shell Integration

- Set default shell (PowerShell or Git Bash per preference)
- Import/configure preferred theme (dark mode to match Samsara aesthetic)
- Set up keybindings for split panes, tab navigation
- Configure Warp AI agent preferences

### 3. Create Samsara Launch Configuration

Create a launch config that opens the standard Samsara dev layout:

- **Pane 1 (main):** `npm start` — Electron main process + Vite dev server
- **Pane 2 (right):** Terminal for Claude Code / agent work
- **Pane 3 (bottom-right):** `cd python-src` — Python sidecar logs / pytest
- **Pane 4 (bottom-left):** Git operations / general terminal

Save as reusable launch configuration for one-click project startup.

### 4. Verify MCP Support

- Check if Warp's MCP integration can connect to any existing MCP servers
- Note any relevant MCP servers for future setup (GitHub, Linear, etc.)

## Success Criteria

- [ ] Warp Terminal installed and launches cleanly on Windows
- [ ] Shell integration works (PowerShell + Git Bash)
- [ ] Samsara launch config opens 4-pane dev layout in one action
- [ ] Claude Code runs inside Warp without issues

## Notes

- Monitor for CPU spike issue reported in Warp GitHub #8409 with Claude Code sessions
- Warp is complementary to Claude Code, not a replacement
