<task type="checkpoint:human-verify" gate="blocking">
  <what-built>
Complete queue management interface with:
- Three-tab queue (Completed/Submitted/Failed)
- Drag-drop CV intake with processing status
- Multi-select with checkboxes and shift-click range
- Bulk retry and delete operations
- Split-view CV editor with inline field editing
- Low-confidence field highlighting (70% threshold)
- Terminal aesthetic (dark, monospace, purple accent)
- Auto-save with debounced IPC
  </what-built>
  <how-to-verify>
**1. Visual Design Check:**
- Launch app with `npm run start`
- Verify: Black background, white text, JetBrains Mono font
- Verify: Samsara header in purple
- Verify: Status badge colors (yellow=submitted, green=completed, red=failed)

**2. Queue Tab Functionality:**
- Verify: Three tabs visible with counts
- Verify: Clicking tab shows filtered items
- Verify: Tab highlight matches status color

**3. Drag-Drop and Processing:**
- Drop a PDF/DOCX file onto drop zone
- Verify: Item appears in Submitted tab
- Verify: Stage text shows (Parsing... -> Extracting... -> Saving...)
- Verify: Item moves to Completed or Failed tab when done

**4. Selection:**
- Click checkbox on an item
- Verify: Item highlighted, count shows in controls
- Shift-click another item
- Verify: Range of items selected
- Click Clear
- Verify: Selection cleared

**5. Bulk Operations:**
- Select multiple completed items
- Click Delete
- Verify: Items removed from queue and database
- (If any failed items exist) Select failed item
- Verify: Retry button appears
- Click Retry
- Verify: Item moves to Submitted, then reprocesses

**6. CV Editor:**
- Click a completed CV
- Verify: Queue shrinks to 50%, editor panel appears on right
- Verify: Contact, Work History, Education, Skills sections visible
- Verify: Confidence badges shown on entries

**7. Inline Editing:**
- Click a field value (e.g., email)
- Verify: Input appears with current value
- Type a new value
- Wait 500ms or click away
- Verify: Value saved (no error)
- Close editor, click same CV
- Verify: Edited value persists

**8. Low-Confidence Highlighting:**
- Find a field with confidence < 70%
- Verify: Field has yellow/warning background and border
- Verify: Hover shows tooltip about low confidence

**9. Close and Navigation:**
- Click Close button in editor
- Verify: Editor panel closes, queue returns to full width
- Click another CV
- Verify: New CV data loads in editor

**Expected outcome:** All checks pass, UI is responsive, no console errors.
  </how-to-verify>
  <resume-signal>Type "approved" if all checks pass, or describe any issues found</resume-signal>
